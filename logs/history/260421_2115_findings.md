## Cambios implementados

### 1. Schema SQL (`db/schema.sql`)

#### Ampliación de `ronda_participantes`
- Agregados campos PT:
  - `participant_code text`: código analítico exportable como `participant_id`
  - `replicate_code integer`: código entero exportable como `replicate`
  - `claimed_at timestamptz`: marca de tiempo de reclamo del cupo
- Constraints nuevos:
  - `unique (ronda_id, participant_code)` para códigos no nulos
  - `unique (ronda_id, replicate_code)` para códigos no nulos
  - `check (replicate_code is null or replicate_code >= 1)`

#### Nueva tabla `ronda_pt_items`
- Configuración PT por contaminante y corrida
- Campos: `id`, `ronda_id`, `contaminante`, `run_code`, `level_label`, `sort_order`, `created_at`
- Constraints:
  - `unique (ronda_id, contaminante, run_code)`
  - `unique (ronda_id, contaminante, level_label)`
  - `unique (ronda_id, contaminante, sort_order)`
  - `check (btrim(run_code) <> '')`
  - `check (btrim(level_label) <> '')`

#### Nueva tabla `ronda_pt_sample_groups`
- Catálogo de grupos de muestra por ronda
- Campos: `id`, `ronda_id`, `sample_group`, `sort_order`, `created_at`
- Constraints:
  - `unique (ronda_id, sample_group)`
  - `unique (ronda_id, sort_order)`
  - `check (btrim(sample_group) <> '')`

#### Nueva tabla `envios_pt`
- Resultados PT capturados por participante
- Campos: `id`, `ronda_id`, `ronda_participante_id`, `pt_item_id`, `sample_group_id`, `mean_value`, `sd_value`, `draft_saved_at`, `final_submitted_at`, `updated_at`
- Constraints:
  - `unique (ronda_participante_id, pt_item_id, sample_group_id)`
  - `check (sd_value >= 0)`
- Índices: `ronda_id`, `ronda_participante_id`, `pt_item_id`
- Trigger: `envios_pt_updated_at` para actualización automática de `updated_at`

## Decisiones técnicas

1. **Separación de identidades**: `workos_user_id` para autenticación, `participant_code` para exportación analítica
2. **Unidad mínima de captura**: `participante + contaminante + run + level + sample_group`
3. **Persistencia incremental**: cada celda PT se guarda independientemente
4. **Envío final**: no crea filas nuevas, solo sella con timestamp todas las filas del participante
