# Technical Finding / Milestone

**Context**: SGC Round Plan Template Blocks (a-u) Integration

## The Discovery

The SGC Round Plan (`F-PPSEA-03`) needs to support editing and viewing all blocks from `a` to `u` in the UI to match the real-world planning template `Planificacion_R1_PP (1).md`. The database and actions layer (`guardarPlanRondaAction`) already supported parsing blocks from `a` to `u`, but the UI only rendered blocks `a` and `b`.

By creating a structured configuration `PLAN_BLOQUES_INFO` listing the descriptions of all 21 blocks, we successfully:
1. Rendered a custom form inside a scrollable viewport constrained to a max-height of `500px` with vertical scrolling. This keeps the form compact and avoids breaking the side-by-side card layout of the SGC page.
2. Formatted the printable plan page (`/dashboard/rondas/[id]/sgc/plan/print`) to dynamically map block keys to their formal template descriptions, presenting them in a clean alphabetical order with fallbacks for empty sections.
