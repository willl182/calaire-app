# Technical Finding / Milestone

**Context**: Directorio de participantes y fichas por ronda

## The Discovery

The participant model can be split cleanly without breaking the existing round flow by introducing a global `directorioParticipantes` table keyed by `nit`, then attaching round slots and fichas to it as optional references. This lets the app precargar basic data for a new round while preserving the historical snapshot of each ficha.

The main operational writes now propagate in two directions:

- updating a ficha's basic participant fields updates the directorio immediately
- creating a ficha seeds its basic fields from the directorio when available
