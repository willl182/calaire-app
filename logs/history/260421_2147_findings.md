# Technical Finding / Milestone

**Context**: Fase 1C completada - Exportador PT compatible con pt_app

## The Discovery

El Carril C de la Fase 1 fue implementado exitosamente, permitiendo la exportación de datos PT en el formato requerido por `pt_app`.

## Implementación realizada

### 1. Tipos PT en `lib/rondas.ts`

Se agregaron los tipos TypeScript necesarios para representar el modelo PT:

- `RondaPTItem`: configuración de corrida y nivel por contaminante
- `RondaPTSampleGroup`: grupos de muestra configurables por ronda
- `RondaParticipantePT`: participantes ampliados con códigos analíticos
- `EnvioPT`: envíos PT transaccionales
- `EnvioPTConMetadatos`: envíos PT con metadatos para exportación

### 2. Funciones de acceso a datos PT

- `listEnviosPTRound(rondaId)`: consulta completa de envíos PT con joins a participantes, items PT y grupos de muestra
- `buildPTCsv(envios)`: generador de CSV PT con el orden exacto de columnas requerido

### 3. Ruta de exportación PT

Se creó `/app/(protected)/dashboard/rondas/[id]/resultados/export-pt.csv/route.ts`:

- Endpoint autenticado para administradores
- Genera CSV PT compatible con `pt_app`
- Nombre de archivo: `{codigo}-pt.csv`
- Orden de columnas: `pollutant`, `run`, `level`, `participant_id`, `replicate`, `sample_group`, `mean_value`, `sd_value`

## CSV generado

El exportador PT genera el formato exacto requerido por `pt_app`:

```
pollutant,run,level,participant_id,replicate,sample_group,mean_value,sd_value
co,run1,low,P001,1,groupA,12.5,0.8
```

## Restricciones observadas

- El exportador depende de que existan datos PT reales en las tablas `envios_pt`
- El modelo SQL de Fase 0 aún no ha sido migrado a `db/schema.sql`
- No hay datos de prueba validados todavía

## Impacto en el plan

- Carril C completado según el plan
- Exportador listo para usar una vez que se complete Carril A (migración SQL)
- Se puede probar con datos PT de prueba cuando existan
