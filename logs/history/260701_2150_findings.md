# Technical Finding / Milestone

**Context**: Code review de Fase 5 (cierre funcional) — calaire-app2 / Recuperacion funcional

## The Discovery

El code review de Fase 5 encontro que el fix de Fase 5 estaba **incompleto**, no defectuoso. Fase 5 corrigio el segmento de API de Convex `sgc:` -> `sgc/index:` (porque `convex/sgc/index.ts` es segmento explicito) **solo** en `scripts/import-sgc-seeds.mjs`.

Un script hermano y **wired** (`pnpm sgc:upload-document-versions` en `package.json:20`), `scripts/upload-sgc-document-versions.mjs`, seguia invocando `sgc:generateUploadUrl`, `sgc:listSgcMaestro` y `sgc:registrarVersionOficial` — el mismo bug "function not found" contra un Convex real. Las tres funciones existen (`convex/sgc/index.ts:27,38,42`). Corregido a `sgc/index:*`.

Regla reutilizable: cuando `convex/<dominio>/index.ts` es segmento explicito, TODO invocador por string (`convex run`, scripts CLI) debe usar `<dominio>/index:<fn>`. Al aplicar un fix de este tipo, hacer `rg "'<dominio>:"` en `scripts/` para no dejar hermanos rotos.

Hallazgos secundarios:
- F32: test E2E acoplado al estado de evidencia (`Registrar evidencia` vs `Reemplazar evidencia`) -> regex tolerante.
- F33: checkboxes de Target 9/10 sin marcar pese al cierre -> marcados con evidencia.
- F34: Target 6 (215 `.collect()` heredados) cerrado por deuda documentada.
- Target 9 cerrado: `diagnostico.md` complementado con bloque de cierre de verificacion (estado RESUELTO).

Deuda detectada: `scripts/poblar-plan-r1.mjs` invoca `sgc:seedPlanRonda`, funcion inexistente en `convex/` y script no wired -> codigo muerto a eliminar.
