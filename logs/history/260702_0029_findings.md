# Technical Finding / Milestone

**Context**: Drive documental SGC / Convex Fase 1

## The Discovery

La Fase 1 puede implementarse como API Convex sin tocar UI ni Google API. El expediente virtual queda persistido en `sgcDriveRecursos`, con una raiz `DRIVE_ROOT`, carpetas por etapa de `SGC_RONDA_ETAPAS` y documentos esperados como hijos.

La inicializacion usa `rondaId + codigo` como clave logica para reutilizar recursos existentes, crear faltantes y reparar metadata de catalogo. No sobrescribe enlaces operativos al reparar notas del catalogo.

Las funciones nuevas incluyen validadores de argumentos y `returns`; en listas/arbol se uso `v.any()` para el shape documental mientras la UI estabiliza el contrato final.
