# Technical Finding — F-PSEA-05A Implementado

**Context**: Digitalización del formulario de registro de participante en CALAIRE-EA

## El Milestone

Implementación completa del formulario F-PSEA-05A en 6 fases: migración SQL, capa DB, server actions, UI del participante, badge en dashboard y vista read-only admin.

## Archivos Creados

- `db/migrate_ficha_registro.sql` — 4 tablas con RLS + trigger + índices
- `lib/fichas.ts` — tipos + 9 funciones (getOrCreateFicha, getFichaByRondaParticipante, getFichaResumenByRondaParticipante, getFichasResumenByRpIds, listFichaResumenesByRonda, upsertFichaScalars, replaceAcompanantes, replaceAnalizadores, replaceInstrumentos, submitFicha)
- `app/(protected)/ronda/[codigo]/registro/actions.ts` — 3 server actions
- `app/(protected)/ronda/[codigo]/registro/page.tsx` — server component
- `app/(protected)/ronda/[codigo]/registro/FormularioRegistro.tsx` — client component
- `app/(protected)/dashboard/rondas/[id]/participantes/[pid]/ficha/page.tsx` — admin read-only

## Archivos Modificados

- `lib/rondas.ts` — `RondaParticipanteAsignada` extendida con `ronda_participante_id` + `ficha_estado`; `listRondasParticipante` hace segunda query a `fichas_registro` para poblar estados
- `app/(protected)/dashboard/page.tsx` — `RondaParticipanteCard` con `FichaBadge` + botón "Iniciar/Continuar ficha"
- `app/(protected)/dashboard/rondas/[id]/participantes/page.tsx` — columna "Ficha" con badge-link, import de `listFichaResumenesByRonda`

## Decisiones de diseño clave

- **Auto-save por campo** (onBlur): llama server action individual, no form submit
- **Listas dinámicas**: estado local en React + botón "Guardar listas" explícito (delete-then-insert)
- **No revalidatePath**: el client component gestiona su estado tras cada acción
- **Allowlist de campos** en `FICHA_SCALAR_ALLOWLIST` — validado en server action, no solo en cliente
- `upsertFichaScalars` filtra `estado = 'borrador'` en la query UPDATE como red de seguridad adicional
- `getOrCreateFicha` hace upsert implícito: select primero, insert si no existe (evita race condition con UNIQUE constraint)
- Vista admin es server component puro (sin `'use client'`), renderiza campos como texto con componente `Campo`
