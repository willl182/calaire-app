# Technical Finding / Milestone

**Context**: Drive documental SGC / Google Drive API

## The Discovery

La fase 5 se implemento mejor como Server Action de Next que llama una capa server-side de Google Drive y luego persiste resultados con los wrappers Convex existentes. Esto evita mezclar runtime Node (`node:crypto` para firmar JWT de cuenta de servicio) en archivos Convex que tambien exportan queries/mutations.

El flujo es idempotente a nivel operativo: si el recurso ya tiene `webUrl`/`driveFolderId` o esta `diligenciado`, `retirado` o `no_aplica`, no se sobrescribe. Ejecutar de nuevo procesa faltantes y recursos pendientes.
