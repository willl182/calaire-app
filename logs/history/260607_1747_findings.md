# Technical Finding / Milestone

**Context**: SGC interface planning for CALAIRE-APP

## The Discovery

The project already supports several SGC-equivalent transactional records through existing app data:

- `rondas` and `rondaContaminantes` cover basic round setup.
- `rondaParticipantes` covers participant assignment, anonymous code and invitation claim state.
- `fichasRegistro` covers participant registration forms.
- `enviosPt` covers PT participant/reference submissions.
- Result pages already calculate basic completitud and CSV exportability.

The missing SGC layer is not a full document manager. The agreed architecture is a round-scoped SGC control panel that combines native records, file evidences and closure gates.

Key architectural decisions:

- Use a new `SGC` tab under each round.
- Add `documentacion_pendiente` as an intermediate round state.
- Treat F-11 as `no_aplica` for this scheme because test items are generated in situ.
- Keep F-08/F-09/F-10/F-14 as file evidences in phase 1.
- Implement the plan of round as editable blocks `a` to `u` based on the provided `F-PPSEA-03` planning template.
- Implement F-13 as a manual checklist with support metrics, not as an automated validation engine.
- Use Convex Storage for SGC evidence files.
- Keep P-20 in phase 1 as Markdown communication templates only.
