# Problem Record (Troubleshooting)

**Severity**: High

## The Problem

Production login and authenticated views failed in sequence:

- WorkOS/AuthKit redirected users back to `http://localhost:3000/auth/callback` after login from production.
- After auth was fixed, the app showed `Backend de Convex no disponible` because Vercel Production had `NEXT_PUBLIC_CONVEX_URL=""`.
- After Convex URL was corrected, `/dashboard` and `/sgc` showed Next error boundary digests because Convex production did not have the public functions the app was calling.

Observed Vercel errors included:

- `Digest: 1732389461` for `GET /dashboard`.
- `Digest: 1214898423` for `GET /sgc`.

Convex logs showed:

- `Could not find public function for 'rondas/index:listRondas'.`
- `Could not find public function for 'rondas/index:listAllParticipantes'.`
- `Could not find public function for 'sgc/index:listSgcMaestro'.`
- `Could not find public function for 'sgc/index:listNormativaSgc'.`
- `Could not find public function for 'sgc/index:listMapaSgc'.`

## User-Requested Corrections

- Fix production login redirecting to localhost.
- Fix production backend connectivity after login.
- Preserve the debugging path and solution using the `saver` skill.

## The Solution / Workaround

1. Updated `src/proxy.ts` to include `/login/start` in `unauthenticatedPaths`.
2. Updated `src/app/login/start/route.ts` to pass an explicit WorkOS `redirectUri` built from forwarded production headers.
3. Added `.vercelignore` to exclude `.env`, `.env.local`, and `.env.*.local` from Vercel CLI deploy uploads.
4. Updated Vercel Production env:
   - Removed empty `NEXT_PUBLIC_CONVEX_URL`.
   - Added `NEXT_PUBLIC_CONVEX_URL=https://steady-kiwi-725.convex.cloud`.
5. Redeployed Vercel production.
6. Deployed Convex production functions with:
   - `CONVEX_DEPLOYMENT=steady-kiwi-725 pnpm exec convex deploy --message "Deploy Convex functions for production app"`

## Learnings to Preserve

- If `/login/start` is protected by AuthKit proxy, the route handler will not run; check `src/proxy.ts` before debugging handler code.
- `request.nextUrl.origin` can be misleading behind middleware/proxy layers; use `x-forwarded-host` and `x-forwarded-proto` when constructing production callback origins.
- `NEXT_PUBLIC_*` env corrections require a Vercel redeploy because client bundles embed those values.
- A Convex URL returning 200 only proves the backend is reachable; missing functions require `convex deploy`.
- `Could not find public function` in Convex logs means the Convex deployment code is stale, not that the app import path is necessarily wrong.
