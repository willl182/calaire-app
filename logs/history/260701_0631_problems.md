# Problem Record (Troubleshooting)

**Severity**: High (bloqueante F19) + Medium (bug funcional en panel admin)

## The Problem

1. **F19 (bloqueante)**: con Convex online, los guards de identidad nuevos rechazaban al usuario autenticado porque `client.ts`/`fichas.ts` no reenviaban el token. Lecturas → fallback vacío sin banner (no era offline); mutaciones → fallaban. Enmascarado porque TODA la verificación de Fase 3 corrió con Convex offline.
2. **Bug funcional (descubierto al hacer F22)**: `dashboard/participantes/[uid]/page.tsx` (admin) llamaba `listRondasParticipante(userId)` para ver las rondas de otro participante, pero el backend derivaba de `identity.subject` e ignoraba el `userId` → devolvía las rondas del admin → `notFound()`. Panel admin de detalle de participante roto desde Fase 3.

## User-Requested Corrections

- "revisa el ultimo rundown e impleementa los fixes de la fase actual" → implementar Fase 3 (F19-F23).
- "no no puede verlos" → un participante NO puede ver resultados agregados de co-participantes (define la política de F20: esos readers → admin-only).
- "corrige todo lo que sea corregible desde ahora" → cerrar también F21 y F22 (y el bug funcional detectado).

## The Solution / Workaround

- F19: wrappers `fetchQuery`/`fetchMutation` que anteponen `{ token }` (patrón `sgcToken()`). Imports reales aliasados a `raw*`.
- F20: guard por-función (admin-only / participant-or-admin-ronda / ownership-ficha).
- F21: `ADMIN_ROLES`/`isAdminRole` centralizados en `sgc/shared.ts`.
- F22 + bug: `listRondasParticipante` con `userId` opcional honrado solo para admins; resto de params self-context eliminados.

## Learnings to Preserve

- **No validar seguridad backend solo con Convex offline**: los fallbacks tragan errores y las mutaciones no se ejercen; F19 pasó desapercibido. Añadir pruebas `convex-test` con `withIdentity` que ejerzan el camino autenticado (éxito propio + rechazo ajeno), no solo el rechazo sin identidad.
- Un "param muerto" puede ocultar un requisito real (admin viendo datos de otro usuario). Antes de borrar, revisar si algún caller le da semántica distinta a la self-context.
- El `agent-router` usa `api.agent.*` (apiKey), no las funciones de dominio; endurecer guards de dominio no lo afecta.
- Endurecer una función consumida por el Convex client de navegador es seguro si la página está gated por rol y hay `ConvexProviderWithAuth` (token WorkOS con el rol).
