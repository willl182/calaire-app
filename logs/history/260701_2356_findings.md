# Technical Finding / Milestone

**Context**: Drive documental SGC por ronda

## The Discovery

El tutorial `t3dotgg/drive-tutorial` no debe portarse literalmente a CALAIRE App. El valor reutilizable es el patrón de producto: árbol de carpetas/archivos, navegación tipo Drive, acciones de gestión y ownership. La implementación local debe usar Convex como backend, WorkOS para auth y el panel SGC de ronda como superficie principal.

El MVP más seguro es crear primero un expediente virtual/manual de enlaces Drive en Convex. La creación automática de carpetas y copias por Google Drive API debe quedar como fase posterior porque depende de decisiones externas: cuenta institucional, cuenta de servicio, OAuth, carpeta raíz y permisos Google Workspace.

## Key Files Affected

- `plan_drive.md`
- `workflow_drive.md`
- `targets_drive.md`
- `convex/schema.ts`

## Implementation Notes

- La UI debe vivir en `/dashboard/rondas/[id]/sgc`.
- No crear `/drive` separado.
- No introducir Drizzle, Prisma, Clerk ni NextAuth.
- Después de tocar Convex se debe correr `pnpm exec convex codegen`.
