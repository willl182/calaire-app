# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

`pnpm test:e2e:start` no quedo verde despues de implementar fase 6. Resultado: 1 prueba paso y 11 fallaron. Las paginas mostraron el overlay de Next con `Runtime TypeError: adapterFn is not a function`, y varias expectativas fallaron porque los encabezados esperados no llegaron a renderizar.

## User-Requested Corrections

- Implementar fase 6 del plan Drive.
- Dejar `drive6.md` con cambios, avances y problemas pendientes.
- Invocar `saver` solo al terminar.

## The Solution / Workaround

- Se confirmo que las validaciones de build si pasan:
  - `pnpm exec convex codegen`
  - `pnpm lint`
  - `pnpm build`
- El fallo E2E quedo documentado como pendiente porque aparece como runtime del entorno de pruebas, no como error de TypeScript o compilacion.

## Learnings to Preserve

- No interpretar este E2E rojo como fallo directo de fase 6 sin revisar primero `adapterFn is not a function`.
- Para cambios App Router en este repo, seguir corriendo `pnpm build`; es la validacion mas estable cuando Playwright queda bloqueado por overlay runtime.
