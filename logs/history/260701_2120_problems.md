# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

Durante Fase 5, `pnpm sgc:import-seeds` fallaba porque el script invocaba `sgc:importarDocumentosSeedSgc`, `sgc:importarRequisitosSeedSgc` y `sgc:importarMapaSeedSgc`. Convex expone esas funciones como `sgc/index:*` porque el repo conserva `convex/sgc/index.ts` como segmento explicito.

Despues, la suite data-backed fallo porque `tests/e2e/sgc-fase2.auth.spec.ts` esperaba UI obsoleta: heading `Panel SGC`, atributo `open` en `#formato-F-PSEA-10` y secciones antiguas de comentarios/notificaciones/pt_app/casos.

## User-Requested Corrections

- Implementar Fase 5 segun `plan_fix.md`, `workflow_fix.md` y `target_fix.md`.
- Dejar `fase5.md` con cambios, avances y problemas pendientes.
- Invocar `saver` solo al terminar.

## The Solution / Workaround

- Se cambio `scripts/import-sgc-seeds.mjs` a `sgc/index:*`.
- Se actualizo `tests/e2e/sgc-fase2.auth.spec.ts` para descubrir ronda real, validar la UI vigente y comprobar foco por `ring-amber-300`.
- Se corrio Convex local como proceso persistente para la suite data-backed y se cerro al terminar.

## Learnings to Preserve

- En este repo, las llamadas CLI a SGC deben usar `sgc/index:<funcion>`.
- `convex dev --once` no reemplaza un backend local vivo para pruebas Next/Playwright.
- Las specs data-backed deben validar contratos actuales accesibles y evitar atributos que la UI no emite.
