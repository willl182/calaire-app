# Technical Finding / Milestone

**Context**: SGC auth migration and security audit

## The Discovery

The app now sends a WorkOS AuthKit access token into Convex through `ConvexProviderWithAuth` and `fetchQuery`/`fetchMutation`, and Convex is configured with `convex/auth.config.ts` for WorkOS JWT validation. That is enough for authenticated transport, but it is not enough for authorization.

The SGC backend in `convex/sgc.ts` is still made of public Convex functions, so each sensitive query and mutation must enforce identity and role checks inside Convex. Otherwise any caller with a valid token, or in the worst case any direct caller, can read or mutate SGC data and forge the `actor` field used in audit rows.

The migration validated cleanly with `pnpm exec tsc --noEmit`, `pnpm lint`, and `pnpm build`.

## Key Files Affected

- `app/providers.tsx`
- `convex/auth.config.ts`
- `lib/sgc/index.ts`
- `convex/sgc.ts`
- `.env.example`
