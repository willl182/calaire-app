# Technical Finding: Primer push a GitHub

**Context**: Configuración de repositorio y control de versiones

## The Discovery

El repo `calaire-app` ya existía en GitHub (`willl182/calaire-app`) pero sin remote configurado localmente. Se resolvió agregando el remote manualmente con `git remote add origin` y luego haciendo push normal. El comando `gh repo create --source . --push` falla si el remote `origin` ya está definido localmente.

Sobre forks: cuando el repo original se actualiza, el fork NO se actualiza automáticamente. Se sincroniza con `git fetch upstream && git merge upstream/main && git push`, configurando el upstream una sola vez.
