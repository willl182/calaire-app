# Problem Record (Troubleshooting)

**Severity**: High

## The Problem

Durante el rediseño del dashboard coordinador, la implementación de acciones por fila quedó incompleta y el usuario reportó que se había quitado el botón de `Reabrir ronda`.

El código sí conservaba `reabrirRondaAction` en `app/(protected)/dashboard/actions.ts`, pero la fila de `RondasTable` no cumplía completamente fase 2.4 porque faltaba el acceso `Resultados` y era necesario confirmar que `StatusAction` siguiera mostrando `Reabrir ronda` cuando `round.estado === 'cerrada'`.

## User-Requested Corrections

- Implementar fase 2.4 del plan `logs/plans/260427_2219_plan_mejoras-ux.md`.
- Restaurar o preservar el botón de `Reabrir ronda`.

## The Solution / Workaround

- Se agregó `RowActionLink` para unificar estilo de enlaces de acción por fila.
- En `RondaRow`, se dejaron visibles `Participantes`, `PT` y `Resultados`.
- Se mantuvo `StatusAction` en la fila; para rondas cerradas renderiza un form hacia `reabrirRondaAction` con el botón `Reabrir ronda`.
- `Editar` queda limitado a rondas en borrador.
- La acción destructiva de borrar ronda quedó separada al final de la fila con estilo rojo.
- Se eliminó el componente muerto `DeleteRondaAction`.
- Se verificó con `npm run build`, que compila correctamente.

## Learnings to Preserve

- En cambios de UX administrativa, nunca retirar acciones de ciclo de vida existentes sin confirmar el reemplazo funcional.
- Para rondas, las transiciones mínimas esperadas son: `borrador -> Publicar`, `activa -> Cerrar`, `cerrada -> Reabrir`.
- La fase 2.4 exige acciones visibles por fila; no basta con que existan server actions.

## Key Files Affected

- `app/(protected)/dashboard/page.tsx`
- `app/(protected)/dashboard/actions.ts`
