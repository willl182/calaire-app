# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

Durante la verificacion inicial, `pnpm build` y `pnpm exec tsc --noEmit` no podian completarse por dos problemas ajenos a los fixes funcionales:

- Faltaba la dependencia directa `@workos-inc/node`.
- El build intentaba descargar fuentes de Google mediante `next/font/google`, bloqueado por el sandbox/red.

## User-Requested Corrections

- Resolver los bloqueos para que `pnpm build` y `pnpm exec tsc --noEmit` pudieran ejecutarse correctamente.
- Guardar el plan inicial de implementacion usando la skill `saver`.

## The Solution / Workaround

- Se agrego `@workos-inc/node` a `package.json` y `pnpm-lock.yaml`.
- Se ejecuto `pnpm install --offline`.
- Se reemplazo el uso de `next/font/google` por stacks locales en `app/layout.tsx` y `app/globals.css`.
- Despues de esos cambios, `pnpm exec tsc --noEmit` y `pnpm build` pasaron.

## Learnings to Preserve

- En este repo, evitar dependencias de red en build, especialmente fuentes remotas.
- Usar `pnpm` para cualquier operacion de dependencias.
- Si un comando critico falla por sandbox/red, reintentar con escalacion siguiendo las reglas del entorno.
