# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

La verificacion Playwright de SGC tuvo tres bloqueos operativos:

1. Chromium de Playwright falla dentro del sandbox:

```text
sandbox_host_linux.cc: Operation not permitted
```

2. El login manual normal de Playwright puede ser rechazado por WorkOS/Google con mensaje de navegador no seguro.

3. Un selector inicial tomo `/dashboard/rondas/nueva` como si fuera ID de ronda real. Al navegar a `/dashboard/rondas/nueva/sgc`, Next mostro error de servidor:

```text
ArgumentValidationError: Value does not match validator.
Path: .id
Value: "nueva"
Validator: v.id("rondas")
```

## User-Requested Corrections

- El usuario indico que para Playwright no se debe insistir en correr dentro del sandbox en este entorno.
- El usuario reporto que el navegador del login manual decia que no era seguro.

## The Solution / Workaround

- Ejecutar Playwright/Chromium fuera del sandbox.
- Para guardar sesion WorkOS, usar Chromium de sistema (`/usr/bin/chromium`) visible con Playwright y argumentos:

```text
--disable-blink-features=AutomationControlled
--start-maximized
```

- Completar login manual ahi y guardar `storageState` en `.auth/workos.json`.
- En verificaciones SGC, excluir explicitamente `/dashboard/rondas/nueva` al descubrir enlaces de ronda:

```ts
href !== "/dashboard/rondas/nueva"
```

o filtrar por patron de ID Convex valido.

## Learnings to Preserve

- En este entorno, Playwright browser launches deben ejecutarse escalados; el sandbox rompe Chromium.
- Si WorkOS/Google muestra "browser no seguro", usar navegador de sistema en vez del Chromium bundled de Playwright.
- `.auth/workos.json` ya existe y debe reutilizarse para pruebas autenticadas.
- No usar `/dashboard/rondas/nueva` como candidato de ronda real.
- Conviene endurecer `/dashboard/rondas/[id]/sgc` para capturar IDs invalidos y devolver 404 limpio antes de llamar funciones Convex que esperan `v.id("rondas")`.
