# Technical Finding / Milestone

**Context**: SGC Maestro CALAIRE protv2 planning

## The Discovery

Se completo un bloque de documentacion de implementacion para convertir el prototipo SGC Maestro en producto real. El plan ahora separa explicitamente tres bloques:

- SGC CALAIRE: repositorio oficial, control documental, versiones, evidencias, matriz normativa y mapa.
- Aplicativo de gestion CALAIRE: rondas, equipos, casos, comunicaciones y operacion interna.
- `pt_app`: sistema externo referenciado, no modulo interno del SGC ni de Gestion.

Tambien se establecio que la implementacion no debe partir de datos vacios ni mocks: debe precargar desde fuentes existentes mediante una etapa intermedia revisable.

## Key Decisions

- Mantener la regla: una sola fuente de datos, varias vistas.
- Precarga auditada antes de importar a Convex.
- `dev/Inventario Documental del SGC.md` es la fuente primaria de documentos maestros PEA.
- `mapa_navegacion_sgc_pea.html` alimenta el mapa SGC vivo y relaciones documentales.
- `dev/req_17043.md` y `dev/req_13528.md` alimentan requisitos normativos detallados.
- ISO/IEC 17025:2017 queda pendiente de fuente equivalente o placeholder aprobado.
- `pt_app` debe manejarse como referencia externa, incluso si existe un documento SGC que lo describe (`DG-PSEA-03`).
