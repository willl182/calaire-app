# Problem Record (Troubleshooting)

**Severity**: High

## The Problem

The production `/dashboard/sgc` route loads its shell, then the page turns black because the client-side Convex query `sgc:listRondasSgcResumen` fails with a `Server Error`.

I verified the failure in two ways:

- Playwright/browser logs showed a `pageerror` for `[CONVEX Q(sgc:listRondasSgcResumen)]`.
- Calling the production Convex deployment directly also returned `Server Error` with a request ID.

The exact backend cause is still unknown because the request ID still needs to be traced inside Convex logs.

## User-Requested Corrections

- Save the session state so the next turn resumes from the unresolved Convex error.
- Preserve the fact that the stable Playwright specs are passing.

## The Solution / Workaround

- The stable parts of the UI were validated and kept in production.
- The query failure was isolated to `sgc:listRondasSgcResumen`.
- The session state now records the remaining blocker and the next debugging target.

## Learnings to Preserve

- A page can look like a client rendering problem when the real issue is a downstream Convex query failure.
- The browser `pageerror` and a direct Convex query call are enough to prove the failure is backend-side.
- When the backend returns a generic `Server Error`, the request ID is the key artifact needed for the next debugging pass.
