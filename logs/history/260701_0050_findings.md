# Technical Finding / Milestone

**Context**: Code review de Fase 3 (Seguridad Convex) — calaire-app

## The Discovery

Fase 3 endurecio `convex/rondas/*`, `convex/pt/index.ts` y `convex/fichas/index.ts` con guards de identidad/rol (`convex/access.ts`) y movio `convex/migrations.ts` a `internalQuery`/`internalMutation`. La revision confirmo:

- **F5 resuelto**: migraciones internas, `wipeAll` incluida, sin consumidores `api.migrations.*` en `src/`. Target 5 cerrado.
- Los guards del backend estan bien escritos y `identity.subject` mapea a `workosUserId` (patron ya usado en `convex/sgc/shared.ts`), por lo que eliminar `userId` como argumento de autorizacion es correcto.
- **Pero** la seguridad no queda funcionalmente verificada: ver el registro de problema asociado (F19), el cliente de servidor no reenvia el token, asi que con Convex online los guards rechazarian al usuario autenticado.

Hallazgos secundarios: F20 (lecturas sensibles solo `requireIdentity` -> fuga cross-lab), F21 (definicion de "admin" divergente entre `access.ts` y `sgc/shared.ts`), F22 (params `userId` muertos en `client.ts`), F23 (test de acceso solo cubre rechazo sin identidad).

## Key Files Affected

- `convex/access.ts`, `convex/rondas/*.ts`, `convex/pt/index.ts`, `convex/fichas/index.ts`, `convex/migrations.ts` (implementacion revisada)
