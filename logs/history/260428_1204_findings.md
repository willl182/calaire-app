# Technical Finding / Milestone

**Context**: Fase 1 — Rediseño UI/UX del dashboard coordinador

## The Discovery

Se completó la reestructuración completa de la navegación y el dashboard del coordinador:

### Patrón: Sidebar como Client Component aislado

El sidebar necesita `usePathname()` + `useSearchParams()` para resaltar el ítem activo. En lugar de convertir todo el layout a Client Component, se extrae el sidebar a un archivo separado `SidebarNav.tsx` con `"use client"`. El layout permanece como Server Component. `useSearchParams()` requiere `<Suspense>` wrapping en Next.js App Router — los exports `Sidebar` y `MobileNav` lo envuelven internamente.

### Patrón: Funciones puras de presentación en `lib/operativo.ts`

Toda la lógica de estado operativo y métricas derivadas vive en un módulo puro sin I/O. Esto permite:
- Reutilización en Fases 2, 3, 4 sin duplicar lógica
- Testing directo sin mocks de Convex
- Separación clara entre datos (queries en page.tsx) y presentación

### Patrón: Menú contextual con `<details>` nativo

Se usa `<details>/<summary>` HTML nativo como menú de acciones secundarias (Editar, Borrar). No requiere JavaScript ni estado. Se cierra en re-render de Server Actions — comportamiento aceptable. Puede refinarse con componente client-side en Fase 5.

### Patrón: KPIs clicables con highlight condicional

Los KPI cards usan `<Link>` en vez de `<div>`, haciéndolos navegables. Cuando un KPI "negativo" (fichas pendientes, cupos sin reclamar) tiene value > 0, se resalta con border/background amber para urgencia visual.
