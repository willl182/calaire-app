# Technical Finding / Milestone

**Context**: Continuacion de implementacion del panel SGC

## The Discovery

Se reviso la deuda de permisos del SGC. Convertir las mutaciones sensibles de `convex/sgc.ts` a `internalMutation` no es compatible con el wrapper actual basado en `fetchMutation` desde server actions, porque `convex/nextjs` espera referencias publicas.

La mitigacion aplicada en esta iteracion fue endurecer las acciones de evidencia:

- `convex/sgc.ts` ahora expone `getEvidenciaVersionContext`.
- `lib/sgc/index.ts` agrega el wrapper server-side correspondiente.
- `app/(protected)/dashboard/rondas/[id]/sgc/actions.ts` valida que una version de evidencia pertenezca a la ronda antes de descargarla o retirarla.

## Validation

- `pnpm exec tsc --noEmit` pasa.
- `pnpm lint` pasa.
- `pnpm build` pasa.

## Remaining Risk

Las mutaciones SGC siguen siendo publicas y aceptan `actor`. La correccion completa requiere migrar a auth Convex real con `convex/auth.config.ts` y `ConvexProviderWithAuth`, o redisenar el canal server-only para que Convex pueda verificar identidad sin argumentos confiados.
