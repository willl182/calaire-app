# Technical Finding / Milestone

**Context**: Complementacion del plan de arreglo funcional de calaire-app (2026-06-30 21:06)

## The Discovery

### 1. `convex run` puede invocar funciones internas â€” el seed no depende de migraciones publicas

`scripts/import-sgc-seeds.mjs` importa datos con `pnpm exec convex run <fn> --identity <json>`, no con `api.migrations.*` del cliente. Verificado con grep: no hay ningun consumidor `api.migrations` en `src/`.

Consecuencia: pasar `wipeAll`, `insertRonda`, `insertFichaRegistro` y las demas ~13 mutaciones de `convex/migrations.ts` a `internalMutation` es seguro y no rompe el seeding. No existe dependencia circular entre Fase 3 y Fase 5 del plan.

Aplicacion futura: cualquier utilidad/carga que exija identidad puede seguir usando `convex run --identity <json>` una vez convertida a interna.

### 2. La reproducibilidad de offline requiere un mecanismo deterministico

Hasta esta sesion, la resiliencia offline de las rutas SGC/rondas/participante solo se verificaba cuando Convex estaba caido por accidente. El mecanismo deterministico es:

```bash
NEXT_PUBLIC_CONVEX_URL=http://127.0.0.1:9 pnpm dev
# o en E2E:
NEXT_PUBLIC_CONVEX_URL=http://127.0.0.1:9 PLAYWRIGHT_START_SERVER=1 playwright test --project=chromium --grep @offline
```

`isConvexOffline` en `src/lib/convex-fallback.ts` ya cubre `ECONNREFUSED`/`fetch failed`, por lo que el mecanismo activa el camino de fallback existente.

### 3. El gate de cierre E2E depende de la disponibilidad de auth WorkOS

El plan anterior no definĂ­a que ocurrĂ­a si las credenciales WorkOS (`AUTH_TEST_*`, `.auth/workos.json`) no estaban disponibles. Decision incorporada:

- Auth disponible â†’ gate = suite autenticada verde.
- Auth ausente â†’ gate degradado = smoke publico verde + bloqueo documentado (no cuenta como fallo de producto).

### 4. `git rm --cached` es prerequisito para dejar de trackear artefactos ya versionados

`docs/screenshots/fase-3/*.png` ya estan en git (aparecen como modificados en `git status`). Agregar `.gitignore` no los excluye retroactivamente; se requiere `git rm --cached <archivo>` primero.
