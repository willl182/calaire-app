# Problem Record: e2e bloqueado por arranque del servidor Playwright

**Severity**: Medium

## The Problem

`pnpm test:e2e` falla porque no arranca servidor y todos los tests reciben `net::ERR_CONNECTION_REFUSED` contra `http://localhost:3000`.

`pnpm test:e2e:start` sí intenta gestionar el servidor, pero el `webServer` agota 120s. Durante el arranque, Next dev emite errores de resolución:

```text
Error: Can't resolve 'tailwindcss' in '/home/w182/w421'
```

El detalle relevante es que `pnpm dev` ejecutado manualmente desde `/home/w182/w421/calaire-app2` arranca limpio en `http://localhost:3000`, así que el fallo parece estar en el entorno/directorio efectivo del `webServer` de Playwright o en cómo Next dev bajo Playwright resuelve CSS.

## The Solution / Workaround

Pendiente. Primer ajuste candidato: añadir `cwd: process.cwd()` al objeto `webServer` de `playwright.config.ts` o ejecutar el comando con cwd explícito, y validar con:

```bash
PLAYWRIGHT_START_SERVER=1 pnpm exec playwright test tests/e2e/app.spec.ts --project=chromium
```

## Learnings to Preserve

- No interpretar este fallo como regresión de rutas de Fase 2 mientras `pnpm build` y `pnpm dev` manual estén verdes.
- El comando `pnpm test:e2e` sin `PLAYWRIGHT_START_SERVER=1` requiere un servidor ya activo.
