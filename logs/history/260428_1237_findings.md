### Regla crítica de Convex — `undefined` en `db.patch`

`ctx.db.patch(id, { campo: undefined })` **elimina el campo opcional** del documento. Esto es comportamiento documentado del SDK de Convex.

La guideline "JavaScript's `undefined` is not a valid Convex value" se refiere a que no se puede **almacenar** o **retornar** `undefined`. No aplica al uso de `undefined` en `patch` para unset de campos opcionales.

El flujo correcto en el código:
```
null (input) → null ?? undefined → undefined → db.patch elimina el campo
```

La sugerencia del reviewer (`if (value !== null) patch.field = value`) **no mejora** la situación: simplemente omite la clave del patch cuando es null, que produce el mismo resultado de no-actualización.
