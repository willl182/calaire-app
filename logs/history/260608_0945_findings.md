# Technical Finding / Milestone

**Context**: Resend con dominio w182.org verificado — smoke test completo de flujo agente

## The Discovery

El dominio `w182.org` verificado en Resend funciona correctamente como remitente (`noreply@w182.org`).
El flujo completo de agente auth funciona end-to-end:

1. `POST /agent/auth` → crea claim + envía email por Resend → `email_delivery.status: "sent"`
2. `POST /agent/auth/claim/complete` con OTP válido → devuelve `api_key` (Bearer)
3. `GET /agent/me` con `Bearer <api_key>` → devuelve `email`, `scopes`, `expires_at`

## Issue Detected

Gmail hace prefetch de links en emails y añade parámetros de tracking (`source=gmail`, `ust`, `usg`) a la URL del `claim_view_url`. Esto puede hacer que el token se pierda o se altere al llegar al servidor. El `page.tsx` de claim view usa `searchParams.token` directamente, pero si Gmail modifica la URL, el token puede no parsearse correctamente.

## Key Files Affected

- `.env.local` — `MAIL_FROM` actualizado a `noreply@w182.org`
- `app/agent/auth/claim/view/page.tsx` — podría necesitar sanitizar query params de Gmail
- `lib/mailer.ts` — Resend con dominio verificado funciona correctamente
