# Technical Finding / Milestone

**Context**: Fase 1 SGC verificada + Agente auth end-to-end + Resend con dominio verificado

## The Discovery

### Fase 1 SGC Completa
- `lib/sgc/catalog.ts` — Catálogo de 12 formatos SGC con fases, modos y criticidad.
- `lib/sgc/checklist.ts` — Lógica de derivación de checklist y bloqueantes desde coverage.
- `convex/schema.ts` — `documentacion_pendiente` en `rondas` + tablas SGC: `sgcPlanRonda`, `sgcRevisionDatos`, `sgcHitosRonda`, `sgcJustificaciones`, `sgcEvidenciaSeries`, `sgcEvidenciaVersiones`, `sgcRegistroSnapshots`, `sgcAuditLog`, `sgcComunicaciones`, `sgcPublicaciones`.
- `convex/rondas.ts` — `assertAllowedEstadoTransition` cubre `borrador → activa → documentacion_pendiente → cerrada` y `cerrada → documentacion_pendiente`.
- `convex/sgc.ts` — 50+ queries/mutations: panel, plan, revisión, hitos, evidencias, justificaciones, comunicaciones, publicaciones, snapshots, audit log.
- `lib/sgc/index.ts` — Cliente SGC con todas las funciones. Se corrigió duplicado `getHitosVisibleParticipante`.

### Agente Auth Funcionando
- `POST /agent/auth` → Resend envía email con `noreply@w182.org` → `email_delivery.status: "sent"`.
- Link del email → WorkOS login → `claim_view_url` muestra OTP de 6 dígitos.
- `POST /agent/auth/claim/complete` con OTP → devuelve `api_key` (Bearer token).
- `GET /agent/me` con Bearer → devuelve `email`, `scopes`, `expires_at`.

### Issue: Gmail Prefetch
Gmail hace prefetch de links en emails y añade parámetros `source=gmail`, `ust`, `usg` a la URL. Esto puede alterar el `token` query param en `claim_view_url`. Se detectó redirección a `/denied?reason=missing_token` cuando Gmail pre-fetch el link.

### Issue: Servidor Colgado
`withAuth({ ensureSignedIn: true })` en `app/agent/auth/claim/view/page.tsx` causaba deadlock en el servidor Next.js. Solución: cambiar a `withAuth()` simple.
