# Technical Finding / Milestone

**Context**: Dashboard tab integration for `Registros`

## The Discovery

`Registros` fits the dashboard's existing navigation model better as a tab than as a standalone page. The consistent implementation is:

1. Expose it as `?tab=registros` in `SidebarNav`.
2. Render the content from `app/(protected)/dashboard/page.tsx` alongside the other tabs.
3. Keep `/dashboard/registros` as a redirect for backward compatibility.

This preserves the existing data sources while making the UI consistent with the rest of the dashboard.

## Key Files Affected

- `app/(protected)/dashboard/page.tsx`
- `app/(protected)/dashboard/SidebarNav.tsx`
- `app/(protected)/dashboard/registros/page.tsx`
- `logs/CURRENT_SESSION.md`
