# Technical Finding / Milestone

**Context**: Integracion entre CALAIRE-EA y `pt_app`

## The Discovery

El archivo `summary_n13.csv` confirma que `pt_app` no consume un consolidado simple por contaminante y nivel. El contrato real exige una fila por:

- `pollutant`
- `run`
- `level`
- `participant_id`
- `replicate`
- `sample_group`

Esto implica que el modelo actual de `calaire-app` no puede resolverse con una exportacion cosmetica del CSV admin existente.

Las decisiones tecnicas cerradas en esta sesion fueron:

- `mean_value` sera calculado en la app.
- `sd_value` se usara como placeholder de incertidumbre.
- `level` debe persistirse como texto.
- `participant_id` debe separarse de `workos_user_id`.

La consecuencia operativa es que primero debe construirse un modelo PT nuevo y despues el exportador compatible.
