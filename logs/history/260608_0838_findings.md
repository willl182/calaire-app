# Technical Finding / Milestone

**Context**: User Claimed / Email Required agent auth flow

## The Discovery

The agent-auth flow works cleanly as a Next.js HTTP surface backed by Convex persistence, with WorkOS/AuthKit only on the human claim view. The key implementation points were:

- `claim_token` and `claim_view_token` are generated separately and only stored as SHA-256 hashes in Convex.
- The human claim view can safely derive OTP display from the `claim_view_token` path and WorkOS email identity.
- Resend is a good fit for this flow because the app can keep `claim_view_url` as a fallback response for development while sending the actual email in production.
- The app remains verifiable in production: `pnpm build` succeeded after the routes were added.
