# Problem Record (Troubleshooting)

**Severity**: High

## The Problem

The new SGC coverage board is implemented, deployed to Vercel production, and the WorkOS session was successfully saved using `/usr/bin/chromium`. However, the production screenshot spec still times out on `/dashboard/sgc` while waiting for the `Resumen SGC` heading:

```text
Test timeout of 30000ms exceeded.
locator.waitFor: waiting for getByRole('heading', { name: 'Resumen SGC' }) to be visible
```

This indicates that the production route is not rendering the expected dashboard markup under Playwright, even with a fresh authenticated session.

## User-Requested Corrections

- Use the system Chromium binary `/usr/bin/chromium`, not the bundled Playwright browser.
- Save the real authenticated session to `.auth/workos.json`.
- Push the change to production and take production screenshots.

## The Solution / Workaround

- Force the manual auth flow to launch `/usr/bin/chromium` with `--disable-blink-features=AutomationControlled` and `--start-maximized`.
- Save the authenticated state to `.auth/workos.json`.
- Re-run the production screenshot spec against `https://calaire-app.vercel.app`.

The session save step succeeded, but the screenshot spec still fails at the route render stage, so the remaining issue is in the production page response rather than browser setup.

## Learnings to Preserve

- The correct browser for WorkOS in this environment is `/usr/bin/chromium`.
- A visible login alone is not enough; the session must be saved to `.auth/workos.json`.
- If production Playwright screenshots still time out after auth is saved, inspect the route response and Vercel logs rather than reworking the browser launcher.

- `tests/e2e/sgc-cobertura-screenshots.auth.spec.ts`
