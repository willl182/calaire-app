# Technical Finding / Milestone

**Context**: SGC Cobertura por Ronda

## The Discovery

The SGC summary page was failing in production because it performed Convex data loading during server-side rendering. Moving the data fetches into a dedicated client component allowed the authenticated page shell to render reliably while `useQuery` handled the data on the client.

For this flow, the following approach proved stable:

- Keep auth/role enforcement in the server page.
- Render a client wrapper for the dashboard content.
- Use authenticated client-side Convex queries for the SGC summary and document matrix.

The validation flow also showed that the Playwright specs must be aligned with the current UI and actual production data:

- The coverage spec must tolerate empty round data.
- The phase 3 screenshot spec must open the `Documentos` tab before asserting the matrix content.
- The current UI does not expose the old `Nuevo documento` / `Registrar version` headings in `/dashboard/sgc`.
