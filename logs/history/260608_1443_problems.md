# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

La verificacion Playwright de Fase 3 no podia cargar `/dashboard/sgc` porque la pagina hacia consultas a Convex y el backend local no estaba escuchando en `127.0.0.1:3210`.

Primero aparecio un error de funciones no publicadas al intentar leer `sgc:listMatrizDocumentalSgc`. Luego, despues de publicar con `convex dev --once`, la verificacion fallo con `ECONNREFUSED 127.0.0.1:3210` porque el backend ya no quedaba persistente.

## User-Requested Corrections

- Tomar varias capturas de Playwright, no una sola.
- Asegurar que la interfaz quede igual a la existente, no simplemente parecida.

## The Solution / Workaround

1. Ejecutar `pnpm exec convex codegen` para regenerar bindings.
2. Ejecutar `pnpm exec convex dev --once` para publicar las funciones nuevas.
3. Mantener `pnpm exec convex dev` corriendo de forma persistente antes de reintentar Playwright.
4. Ejecutar Playwright fuera del sandbox, porque Chromium falla dentro del sandbox con `sandbox_host_linux.cc: Operation not permitted`.

## Learnings to Preserve

- `convex dev --once` es util para publicar, pero no deja el backend listo para una sesion Playwright posterior.
- Si la ruta Next usa Convex en render server-side, el backend local debe estar vivo durante toda la prueba.
- Para este repo, las capturas de validacion deben dividirse por verificacion: resumen global, matriz documental, formulario de documento y formulario de version.
