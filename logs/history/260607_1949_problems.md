# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

The SGC page visually diverged from the other round tabs. The initial issue was interpreted as tab width, but the user screenshots showed a broader design mismatch:

- The SGC page wrapper/header looked narrower or inconsistent compared with Resumen and Configuracion PT.
- The SGC top card used ad-hoc classes: `rounded-xl border border-[var(--border)] bg-white p-6 shadow-sm`.
- Existing tabs use shared global classes: `header-bar px-6 py-5` for the top round header and `card p-6` for content cards.

## User-Requested Corrections

- Make SGC the same size as the other tabs.
- Make SGC cards homogeneous with the existing tab style.
- Deploy the correction to production, not sandbox-only.

## The Solution / Workaround

Updated `app/(protected)/dashboard/rondas/[id]/sgc/page.tsx`:

- Page wrapper now matches other tabs:
  - `div className="min-h-screen bg-[var(--background)] px-6 py-8"`
  - inner `div className="mx-auto flex max-w-7xl flex-col gap-6"`
- SGC top panel now uses:
  - `header className="header-bar px-6 py-5"`
- SGC sections now use:
  - `section className="card p-6"`
- Alerts now sit after the header, matching the pattern used in other round pages.

Validated with:

- `pnpm exec tsc --noEmit`
- `pnpm lint`

Deployed with:

- `vercel --prod`

Latest production deployment:

- Deployment ID: `dpl_FgRYmUwqpttQreQ2FXSqFmjcYvUM`
- Alias: `https://calaire-app.vercel.app`

## Learnings to Preserve

- When the user says a tab is "smaller" or "not homogeneous", compare the whole page wrapper and shared design classes, not only the tab item width.
- For this app, round subpages should use the established pattern:
  - outer background/padding wrapper
  - `mx-auto flex max-w-7xl flex-col gap-6`
  - `RondaContextNav`
  - `header-bar px-6 py-5`
  - `card p-6`
- Avoid ad-hoc `rounded-xl border bg-white shadow-sm` wrappers when an existing design-system class exists.
