# Technical Finding: API key admin emitida para Agent API

**Context**: Se completo el flujo de claim para permitir que el agente use `/agent/v1/*` con permisos admin.

## The Discovery

- Se creo una claim para `wilsonsalasc@gmail.com` usando `POST /agent/auth`.
- Resend reporto envio exitoso del correo de claim.
- El usuario abrio la URL de claim y entrego el OTP.
- Se completo la claim con `POST /agent/auth/claim/complete`.
- La API key nueva fue verificada contra `GET /agent/me`.
- Los scopes verificados fueron `calaire.agent.me` y `calaire.agent.admin`.

## Important Clarification

- La API key es para acceso programatico por `Authorization: Bearer <key>`.
- La API key no equivale a una sesion web WorkOS.
- Para tomar pantallazos de paginas protegidas como `/dashboard`, Playwright necesita una sesion WorkOS real. El flujo recomendado es login interactivo una vez y guardar `storageState`.
