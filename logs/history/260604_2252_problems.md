# Problem Record (Troubleshooting)

**Severity**: High

## The Problem

The `/dashboard` route broke after introducing the participant directory UI. The new panel mixed `directorioParticipantes` fields with `ParticipanteGlobal`, but the aggregated global participant type does not expose the directory-only fields. This caused TypeScript build failures and a broken production render path while iterating on the admin screen.

## User-Requested Corrections

- Restore `/dashboard` to a working state.
- Keep the admin panel stable rather than forcing directory fields through the wrong aggregate type.
- Preserve the production deployment after the fix.

## The Solution / Workaround

- Simplified the directory panel to consume only `ParticipanteGlobal` data.
- Removed the row-level directory table that depended on fields not present in the aggregate type.
- Kept the direct directory creation form wired through `app/(protected)/dashboard/participantes/actions.ts`.
- Re-ran `pnpm lint` and `pnpm build` successfully.
- Redeployed to production.

## Learnings to Preserve

- Do not project directory-specific columns from `ParticipanteGlobal` unless the backend type explicitly exposes them.
- When changing the dashboard, keep the admin panel aligned with the exact backend contract before adding richer UI controls.
- A working build locally is necessary but not sufficient; verify the route shape against the real data contract before redeploying.
