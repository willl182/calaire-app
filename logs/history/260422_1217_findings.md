# Technical Finding / Milestone

**Context**: Gestión de perfiles de participantes por ronda (referencia)

## The Discovery

Se implementó un tercer perfil de participante para referencias PT: `member_special`, coexistiendo con `member`.

Capacidades entregadas:
- Perfil persistente por participante en `ronda_participantes.participant_profile`.
- Generación de enlace individual para referencia (slot pendiente con token), igual que para participantes.
- Botón específico para agregar referencia desde la vista de participantes.
- Opción en creación de ronda para incluir referencia automáticamente (`include_reference`).
- Validación de seguridad en claim: solo cuenta con rol `member special` puede reclamar enlace de referencia.
- Mensaje explícito de denegación para mismatch de rol en enlaces de referencia.

Decisiones técnicas clave:
- Índice único parcial para asegurar máximo 1 referencia por ronda.
- Compatibilidad de rol con alias: `member_special`, `member-special`, `member special`.
- Separación clara de etiquetas de cupo pendiente entre participante y referencia.
