# Technical Finding / Milestone

**Context**: Panel de ficha del participante

## The Discovery

La pantalla `/ronda/[codigo]/registro` dejaba al participante dentro de la ficha sin controles visibles para volver al dashboard, cerrar sesión o avanzar a carga de datos cuando correspondiera. Esto rompía el flujo operativo después de enviar la ficha.

## Implementation

Se agregó una zona de acciones en el header de `FormularioRegistro`:

- `Volver` enlaza a `/dashboard`.
- `Cargar datos` enlaza a `/ronda/[codigo]` solo si la ronda está activa y la ficha ya fue enviada.
- Si no se puede cargar datos, se muestra como estado deshabilitado con tooltip contextual.
- `Cerrar sesión` usa una server action local: `cerrarSesionParticipanteAction`.

Para poder decidir si `Cargar datos` está activo, `registro/page.tsx` pasa `ronda.estado` al componente cliente como `rondaEstado`.

## Key Files Affected

- `app/(protected)/ronda/[codigo]/registro/FormularioRegistro.tsx`
- `app/(protected)/ronda/[codigo]/registro/actions.ts`
- `app/(protected)/ronda/[codigo]/registro/page.tsx`

## Verification

`npm run build` pasó correctamente después del ajuste.
