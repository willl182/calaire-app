# Technical Finding / Milestone: Continuacion SGC Maestro protv2

**Context**: Implementacion SGC Maestro CALAIRE y separacion de dashboards.

## The Discovery

La separacion correcta no es solo conceptual, tambien debe reflejarse en rutas, navegacion y documentacion:

- SGC maestro global controla documentos, versiones, normativa y mapa.
- Dashboard documental por ronda pertenece a Gestion/Rondas y evidencia el cierre operativo de una ronda.
- `/dashboard/sgc/expedientes` no debe figurar como ruta real del SGC maestro; queda como redireccion de compatibilidad.

## Work Completed

- Se elimino el resumen legacy de rondas dentro de `/dashboard/sgc`.
- Se removieron componentes legacy no importados que mezclaban cobertura de rondas en SGC maestro.
- Se agrego descarga de version oficial desde detalle de documento.
- Se agrego relacion documento-requisito desde la matriz normativa.
- Se agregaron helpers de permisos SGC:
  - `canViewSgcMaestro`
  - `canEditSgcMaestro`
- Se actualizaron `dev/plan-protv2.md`, `dev/workflow-protv2.md`, `dev/targets-protv2.md` para reflejar la separacion.
