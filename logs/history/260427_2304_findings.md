# Technical Finding: KpiBar — Fase 2.2 / 2.6

**Context**: Dashboard coordinador — métricas globales del programa

## The Discovery

### Cálculo de fichas pendientes sin nueva query Convex

Para obtener el conteo de fichas pendientes en rondas activas se reutilizó `listParticipantesRondaResumen` (que ya existía) en paralelo sobre cada ronda activa:

```ts
const rondasActivas = rondas.filter((r) => r.estado === 'activa')
const participantesRondasActivas = admin
  ? await Promise.all(rondasActivas.map((r) => listParticipantesRondaResumen(r.id)))
  : []
const fichasPendientesCount = participantesRondasActivas
  .flat()
  .filter((p) => p.ficha_estado !== 'enviado').length
```

Esto evita añadir una nueva query Convex. El costo es 1 query por ronda activa en cada render del dashboard admin.

### `listAllParticipantes()` ahora incondicional

Antes: `admin && activeTab === 'participantes' ? await listAllParticipantes() : []`
Ahora: `admin ? await listAllParticipantes() : []`

Esto permite alimentar el KpiBar con `totalParticipantes` y mostrar el badge de conteo en `TabNav` en ambos tabs.

### `.card-accent` ya existía

El plan mencionaba `.card-accent` (borde dorado izquierdo). La clase ya estaba definida en `globals.css`:
```css
.card-accent {
  border-left: 4px solid var(--pt-primary);
  border-radius: 12px;
  ...
}
```
No hubo que crear estilos nuevos.
