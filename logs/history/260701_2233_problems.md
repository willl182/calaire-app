# Problem Record (Troubleshooting)

**Severity**: High (bloqueaba el deploy del PR)

## The Problem

El build en Vercel (rama `feature/t3-estructura-segura`, commit b0bc12a) fallaba en la fase "Collecting page data":

```
Invalid environment variables: [
  WORKOS_API_KEY: expected string, received undefined
  WORKOS_CLIENT_ID: expected string, received undefined
  NEXT_PUBLIC_CONVEX_URL: expected string, received undefined
]
Error: Failed to collect page data for /agent/auth
```

Causa: `src/env.ts` usa `@t3-oss/env-nextjs` + zod y valida las vars al importarse. `next build` importa `env.ts` al recolectar `/agent/auth`; como las vars no estaban configuradas en el proyecto Vercel, zod aborta. NO era un bug de codigo: `env.ts` estaba correcto.

## User-Requested Corrections

- El usuario pidio que YO empujara las variables via CLI (opcion "CLI: yo las empujo") desde `.env.local`.

## The Solution / Workaround

1. `vercel whoami` -> autenticado como `willl182`; proyecto ya linkeado en `.vercel/project.json` (`calaire-app`).
2. Cargar en Production leyendo valores de `.env.local`:
   `printf '%s' "$VAL" | vercel env add "$V" production` (previo `vercel env rm "$V" production -y` para evitar prompts).
3. Preview fallaba: el CLI 52.2.1 devuelve `git_branch_required` incluso con `--value --yes`. Workaround: fijar la rama:
   `vercel env add "$V" preview feature/t3-estructura-segura --value "$VAL" --yes`.
4. Verificado con `vercel env ls`: 7 vars en Production + 7 en Preview(rama).

Vars cargadas: WORKOS_API_KEY, WORKOS_CLIENT_ID, WORKOS_COOKIE_PASSWORD, NEXT_PUBLIC_WORKOS_REDIRECT_URI, NEXT_PUBLIC_CONVEX_URL, RESEND_API_KEY, MAIL_FROM.

## Learnings to Preserve

- Cualquier build de este repo en Vercel requiere al menos las 3 vars requeridas por `env.ts` (WORKOS_API_KEY, WORKOS_CLIENT_ID, NEXT_PUBLIC_CONVEX_URL) presentes en el entorno correspondiente (Production y/o Preview).
- El CLI Vercel 52.2.1 esta desactualizado y no soporta bien Preview "all branches" no interactivo; actualizar a >=54 o usar el dashboard. Mientras tanto, cargar Preview por rama.
- `.env.local` usa `WORKOS_COOKIE_PASSWORD` (no `WORKOS_SECRET`). WorkOS authkit-nextjs necesita el cookie password en runtime.
- Preview por-rama solo cubre esa rama; nuevos PRs desde otras ramas necesitaran repetir la carga o configurar "All Preview".
