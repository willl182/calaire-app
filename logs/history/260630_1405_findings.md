# Technical Finding / Milestone

**Context**: Migración T3 / Convex auth config / cierre de Fase 8

## The Discovery

`convex/auth.config.ts` estaba importando `src/env.ts`, lo que forzaba a Convex a validar todas las variables declaradas en `createEnv` durante `pnpm exec convex codegen` y `pnpm exec convex dev`. Aunque `auth.config.ts` solo usaba `WORKOS_CLIENT_ID`, el import hacía fallar Convex si `WORKOS_API_KEY` no existía en el entorno del deployment usado por Convex.

La corrección fue desacoplar `convex/auth.config.ts` de `src/env.ts` y leer únicamente `process.env.WORKOS_CLIENT_ID` con una validación explícita local. Con eso:

- `WORKOS_API_KEY` sigue existiendo para runtime de la app.
- Convex deja de exigir variables que `auth.config.ts` no usa realmente.
- `pnpm exec convex codegen` vuelve a pasar.

También quedó actualizada la documentación de Fase 8:

- `README.md` refleja la estructura T3 actual.
- `AGENTS.md` documenta paths, capas y verificación.
- `.env.example` quedó alineado con `src/env.ts`.
