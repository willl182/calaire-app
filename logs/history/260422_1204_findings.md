# Technical Finding / Milestone

**Context**: Ampliación del modelo de captura PT — 7 campos por celda

## The Discovery

El formulario de captura PT pasó de 2 campos por celda (mean + sd) a 7:
- `d1`, `d2`, `d3`: datos individuales (pueden ser negativos)
- `mean_value`: promedio de los 3
- `sd_value`: desviación estándar (≥ 0)
- `ux`: incertidumbre estimada del promedio (≥ 0)
- `ux_exp`: incertidumbre expandida (≥ 0)

Los nuevos campos en DB son nullable para no romper filas existentes. La validación de completitud se hace en la capa de aplicación antes del autoguardado.

El rename de columnas (`u_mean`→`ux`, `u_expandida`→`ux_exp`) se hace en el mismo script de migración con `ALTER TABLE ... RENAME COLUMN`.
