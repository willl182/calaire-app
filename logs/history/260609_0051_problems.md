# Problem Record (Troubleshooting)

**Severity**: High

## The Problem

The SGC summary route `/dashboard/sgc` returned a production server error under Playwright, so screenshots timed out waiting for `Resumen SGC`.

The failure chain was:

- SSR on the Next.js page called Convex queries directly.
- Production rendered a server error page instead of the dashboard shell.
- The screenshot spec timed out waiting for the heading.

Later, the functional and screenshot specs also exposed brittle assumptions:

- The functional spec assumed production data contained a round code matching `R-`.
- The phase 3 screenshot spec assumed the old `Nuevo documento` / `Registrar version` UI still existed.
- The phase 3 screenshot spec also needed to open the `Documentos` tab explicitly.

## User-Requested Corrections

- Deploy the fix to production.
- Re-run the Playwright checks.
- Use `/usr/bin/chromium` instead of the bundled Playwright browser.
- Avoid running Chromium validation from the sandbox when it blocks browser startup.

## The Solution / Workaround

- Moved the SGC summary data loading into `app/(protected)/dashboard/sgc/SgcResumenClient.tsx`.
- Kept auth/role enforcement in `app/(protected)/dashboard/sgc/page.tsx`.
- Updated `convex/sgc.ts` to require authentication for the summary queries without duplicating the page-level role gate.
- Updated Playwright config to launch `/usr/bin/chromium`.
- Made the functional spec tolerant of empty production data.
- Reworked the phase 3 screenshot spec to follow the current UI flow and current headings.

## Learnings to Preserve

- If a server component crashes before the heading renders, the screenshot timeout is a symptom, not the root cause.
- When production data may be empty, specs should assert the UI state, not hard-coded fixture values.
- Playwright selectors must match the rendered role and exact UI text, especially after tab-based layouts change.
- For this environment, Chromium validation that needs a real browser should be run outside the sandbox when the sandbox blocks startup.
