# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

Al ejecutar la suite completa de pruebas E2E de Playwright en paralelo (por defecto con múltiples workers), los tests fallaban de manera consistente por timeouts (`Test timeout of 30000ms exceeded`). Los navegadores no lograban cargar o verificar elementos a tiempo debido a la contención en el servidor Next.js local y la base de datos Convex que corren dentro del sandbox.

## User-Requested Corrections

- Poder visualizar los reportes locales exitosos en la URL `http://localhost:9323/`.

## The Solution / Workaround

1. **Ejecución secuencial:** Correr la suite de pruebas de Playwright forzando el uso de un solo worker:
   ```bash
   PLAYWRIGHT_BASE_URL=http://localhost:3000 pnpm exec playwright test --project=authenticated-chromium --workers=1
   ```
   Esto reduce el consumo de CPU y memoria, eliminando los timeouts y permitiendo que todos los tests pasen exitosamente (11.1 segundos en total).

2. **Levantar el servidor de reporte:** Exponer el reporte de Playwright en la red local del sandbox:
   ```bash
   pnpm exec playwright show-report --host 0.0.0.0 --port 9323
   ```

## Learnings to Preserve

- En entornos restringidos de contenedores/sandbox, el paralelismo en Playwright (`workers > 1`) puede generar cuellos de botella severos de I/O y CPU en el servidor Next.js de desarrollo.
- Siempre preferir ejecutar con `--workers=1` para validaciones robustas y estables de e2e en local.
