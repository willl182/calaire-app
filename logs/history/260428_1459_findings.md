# Technical Finding / Milestone

**Context**: UX de configuración PT y flujo de participante

## The Discovery

El textarea para cargar niveles PT era rápido técnicamente, pero incómodo para usuarios operativos. La pantalla necesitaba una forma estructurada similar a la carga de datos: filas explícitas con campos editables y controles para agregar más filas.

También se detectó que el dashboard de participante mostraba “Próximamente” en rondas en borrador y no ofrecía el botón para diligenciar ficha, aunque el flujo deseado es que el participante pueda completar la ficha antes de que el coordinador active la ronda.

## Implementation

- Se creó `PTLevelsBulkForm.tsx` como componente cliente.
- La UI ahora usa una grilla con columnas `Run` y `Nivel`.
- Los botones `+ 1 fila` y `+ 5 filas` agregan filas sin recargar.
- La sección “Alta rápida” fue eliminada.
- El dashboard de participante siempre muestra acción de ficha.
- `Cargar datos` se muestra como acción separada, pero solo queda activa cuando la ronda está activa y la ficha enviada.
- La ruta `/ronda/[codigo]/registro` ya no bloquea fichas en rondas borrador para participantes invitados.

## Key Files Affected

- `app/(protected)/dashboard/rondas/[id]/configuracion-pt/PTLevelsBulkForm.tsx`
- `app/(protected)/dashboard/rondas/[id]/configuracion-pt/page.tsx`
- `app/(protected)/dashboard/page.tsx`
- `app/(protected)/ronda/[codigo]/registro/page.tsx`

## Verification

`npm run build` pasó correctamente después de los cambios.
