# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

Dos problemas aparecieron durante la implementación de Playwright:

1. `pnpm test:e2e` falló inicialmente porque `playwright.config.ts` intentaba iniciar `pnpm dev`, pero ya existía un `next dev` activo en `http://localhost:3000`. Next 16 reportó: "Another next dev server is already running".
2. Chromium de Playwright falló dentro del sandbox con `FATAL:content/browser/sandbox_host_linux.cc:41` y `Operation not permitted`.

## User-Requested Corrections

- El usuario aclaró que no necesita login automatizado para producción.
- El usuario aceptó hacer login manual si eso permite que el agente pueda ingresar al aplicativo local.

## The Solution / Workaround

- `pnpm test:e2e` ahora corre contra un servidor ya activo en `http://127.0.0.1:3000`.
- `pnpm test:e2e:start` queda como modo separado para iniciar servidor cuando sea necesario.
- Se agregó `pnpm test:e2e:auth:manual`, que abre Chromium visible y guarda `storageState` después de llegar a `/dashboard`.
- `.auth/workos.json` se ignora en git y se reutiliza para pruebas autenticadas.
- Para correr Chromium en este entorno se usó ejecución fuera del sandbox; `pnpm test:e2e` quedó aprobado.

## Learnings to Preserve

- No intentar iniciar un segundo `next dev` para este proyecto si ya hay uno activo en puerto 3000.
- Para WorkOS/AuthKit local, preferir sesión manual guardada cuando MFA/captcha o email links hagan frágil el login automatizado.
- Las pruebas autenticadas deben depender de `.auth/workos.json` y no romper el smoke público cuando ese archivo no exista.
