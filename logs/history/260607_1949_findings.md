# Technical Finding / Milestone

**Context**: SGC implementation and production deployment

## The Discovery

The SGC feature is implemented as a round-scoped admin panel with Next.js server actions calling Convex through server-side wrappers. Since the app authentication is handled by WorkOS/Next, SGC Convex functions use a shared `serverSecret` to prevent direct unauthenticated calls to public Convex functions.

The SGC route is available at `/dashboard/rondas/[id]/sgc` and includes checklist, closure actions, plan, F-PSEA-13, evidence management, cronograma, templates, and audit log sections.
