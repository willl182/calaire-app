# Technical Finding / Milestone

**Context**: CALAIRE-APP como interfaz operativa del SGC

## The Discovery

Se cerro una sesion de diseno tipo `grill-me` para definir el alcance del panel SGC.

La decision central es que CALAIRE-APP no debe convertirse en un repositorio documental completo. Debe operar como panel de cierre documental por ronda, gestionando de forma nativa solo los registros vivos, trazables y operativos: checklist, cronograma, evidencias, comunicaciones, notificaciones y comentarios.

Los documentos narrativos, procedimientos, instructivos, matrices, informes externos y soportes generados por otras herramientas deben conservarse como archivos/evidencias, con versionamiento sencillo cuando aplique.

## Key Decisions

| Tema | Decision |
|---|---|
| Ubicacion | Nueva pestaña `SGC`; no absorber `Registros` |
| Eje | Ronda |
| MVP 1 | Checklist, hitos, evidencias, comunicaciones/notificaciones, comentarios |
| MVP 2 | Formularios nativos SGC, casos formales, importacion estructurada `pt_app` |
| Participantes | Ven solo hitos/notificaciones/comentarios/publicaciones que les correspondan |
| Cronograma | Tabla de hitos minimos; no calendario/Gantt inicial |
| Comunicaciones | Preparar/registrar manualmente; no automatizar email en MVP |
| Evidencias | Serie documental + versiones, con una version vigente |

## Key Files Affected

- `plan_codex.md`: plan de implementacion y arquitectura funcional.
- `grillme-codex.md`: acta de la sesion `grill-me` con preguntas, recomendaciones, respuestas y decisiones.
- `logs/CURRENT_SESSION.md`: estado actual para continuidad.

## Implementation Notes

Si se implementa el MVP 1, el orden recomendado es:

1. Leer `convex/_generated/ai/guidelines.md`.
2. Crear tablas Convex SGC MVP 1.
3. Agregar pestaña `SGC` al dashboard admin.
4. Crear panel por ronda con checklist y cronograma.
5. Agregar evidencias versionadas.
6. Agregar comunicaciones/notificaciones.
7. Agregar comentarios de ronda.
8. Exponer piezas limitadas al participante en `Mi dashboard`.
