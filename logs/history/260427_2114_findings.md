# Technical Finding — Fase 6: Supabase removido de la app

**Context**: Migracion Supabase -> Convex. Cierre de limpieza despues de migrar datos a Convex prod.

## Resultado

La app ya no usa Supabase en runtime. Las Server Actions administrativas que aun llamaban `getSupabaseAdmin()` fueron reescritas para usar la capa Convex.

## Cambios clave

- `convex/rondas.ts`: nuevas mutaciones transaccionales para crear/editar/eliminar rondas, cambiar estado, administrar participantes y manejar cascadas manuales.
- `lib/rondas.ts`: wrappers server-side con `fetchMutation` para las nuevas mutaciones.
- `app/(protected)/dashboard/actions.ts`: reemplazo de CRUD de rondas Supabase por Convex.
- `app/(protected)/dashboard/rondas/[id]/participantes/actions.ts`: reemplazo de alta/regeneracion/eliminacion de participantes por Convex.
- `app/(protected)/dashboard/rondas/[id]/configuracion-pt/actions.ts`: reemplazo de checks PT y borrados por helpers Convex.
- `app/(protected)/dashboard/rondas/[id]/participantes/[pid]/ficha/actions.ts`: guardado admin de campo de ficha via `upsertFichaScalars`.
- `lib/supabase.ts`: eliminado.
- `@supabase/supabase-js`: removido de `package.json` y `package-lock.json`.
- `scripts/migrate-supabase-to-convex.mjs`: eliminado tras completar la migracion operativa.
