# Technical Finding / Milestone

**Context**: Automated release flow for CALAIRE App

## The Discovery

The repo did not have a single release entrypoint. The correct operational order for this project is:

1. run `pnpm lint`
2. run `pnpm build`
3. stage and commit the changes
4. deploy Convex first
5. deploy Vercel after Convex

That order matters because the frontend can depend on newly deployed Convex functions/schema.

## Key Files Affected

- `package.json`
- `scripts/release.mjs`
- `README.md`
