# Technical Finding — Migración envios_pt completada

**Context**: Migración `migrate_envios_pt_nuevos_campos.sql` en Supabase

## El Problema

La migración original hacía `add column if not exists u_mean/u_expandida` y luego `rename column u_mean → ux`. Al correrla por segunda vez falló con `ERROR: 42701: column "ux" already exists` porque las columnas ya habían sido renombradas en una ejecución previa parcial.

## La Solución

Correr solo el `add column if not exists` con los nombres finales directamente:

```sql
alter table envios_pt
  add column if not exists d1     double precision,
  add column if not exists d2     double precision,
  add column if not exists d3     double precision,
  add column if not exists ux     double precision,
  add column if not exists ux_exp double precision;
```

## Estado final en Supabase

Tabla `envios_pt` tiene: `d1`, `d2`, `d3`, `ux`, `ux_exp` — todos presentes y listos para captura PT.

## Lección

Cuando una migración con `rename column` falla por ejecución parcial previa, no re-correr el rename — usar `add column if not exists` con el nombre final.
