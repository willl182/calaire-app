# Technical Finding / Milestone

**Context**: Agent auth smoke test with Resend and AuthKit proxy

## The Discovery

The agent auth API endpoints must bypass the global AuthKit `proxy`, while the human claim page must remain covered by it. Excluding all of `/agent/*` makes `/agent/auth/claim/view` fail because `withAuth` requires AuthKit proxy coverage. The correct boundary is to exclude only exact API endpoints:

- `/agent/auth`
- `/agent/auth/claim`
- `/agent/auth/claim/complete`
- `/agent/me`

Resend can be configured correctly at the API-key level while still rejecting delivery if `MAIL_FROM` uses an unverified domain. The route should not treat that as a failed claim creation; returning `claim_view_url` preserves the development and agent handoff flow.

## Key Files Affected

- `proxy.ts`
- `lib/mailer.ts`
- `app/agent/auth/route.ts`
- `app/agent/auth/claim/route.ts`
