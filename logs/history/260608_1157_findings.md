# Technical Finding: Agent REST API para Calaire App

**Context**: Implementación de endpoints REST para que un agente autenticado (vía API key del flujo de claim) pueda operar como admin dentro de la app.

## The Discovery

### Problema inicial
El agente ya tenía un endpoint `/agent/me` que validaba la API key, pero no había endpoints que permitieran al agente hacer operaciones en la app. Las funciones Convex existentes (`rondas.ts`, `sgc.ts`, `pt.ts`, `fichas.ts`) usan `ctx.auth.getUserIdentity()` que depende de WorkOS — el agente no tiene sesión WorkOS.

### Solución
1. **Nuevo módulo `convex/agent.ts`**: 40+ funciones que reimplementan o envuelven la lógica de admin, pero validan autenticación con `requireAgentAuth(apiKey)` / `requireAgentAdmin(apiKey)` que verifica el hash SHA-256 de la API key contra la tabla `agentApiKeys`.

2. **Router REST `lib/agent-router.ts`**: tabla `ROUTES` que define cada endpoint con:
   - `method`: HTTP method
   - `pattern`: patrón de URL (ej: `rondas/:id/sgc`)
   - `convexFn`: nombre de la función en `convex/agent.ts`
   - `kind`: `query` o `mutation`
   - `extractArgs`: función que extrae parámetros de la URL
   - `bodyFields`: campos que se extraen del body JSON

3. **Endpoint Next.js `app/agent/v1/[...path]/route.ts`**: captura cualquier path bajo `/agent/v1/*`, extrae `Authorization: Bearer <api_key>`, busca la ruta en `ROUTES`, y llama a Convex con los args correctos.

4. **Scope `calaire.agent.admin`**: agregado automáticamente a nuevas claims en `lib/agent-auth.ts`. Las funciones de escritura lo requieren; las de lectura solo necesitan `calaire.agent.me`.

### Validación
- Build: `pnpm run build` ✅
- `GET /agent/v1/rondas` con Bearer key → devuelve lista de rondas ✅
- `GET /agent/v1/rondas/:id/sgc` con Bearer key → `Scope calaire.agent.admin requerido` (comportamiento esperado, la key antigua no tiene ese scope) ✅

## Key Files Affected

- `convex/agent.ts` — nuevo, 40+ funciones para agente
- `lib/agent-router.ts` — nuevo, mapeo de rutas REST a funciones Convex
- `app/agent/v1/[...path]/route.ts` — nuevo, endpoint REST genérico
- `lib/agent-auth.ts` — modificado, scope `calaire.agent.admin` añadido
- `proxy.ts` — modificado, regex de matcher actualizado para no interceptar `/agent/v1/*`
- `convex/_generated/api.d.ts` — regenerado, incluye módulo `agent`
