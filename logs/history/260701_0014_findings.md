# Technical Finding / Milestone

**Context**: Cierre Fase 2 — UX offline y estados vacios

## The Discovery

El antipatron de doble captura seguia vivo fuera de SGC: `safeConvexCall` devolvia fallback vacio y los `try/catch` superiores nunca recibian el error offline. La solucion consistente fue extender variantes `WithStatus` y derivar `backendOffline` desde `.offline`.

## What Changed

- `/dashboard` admin usa loaders `WithStatus` para rondas, participantes, resumen de participantes y resultados PT.
- `/mi-dashboard` usa lecturas SGC `WithStatus` y no muestra "sin rondas asignadas" cuando Convex esta offline.
- `/dashboard/rondas/[id]` usa `getRondaMetricasCompletasWithStatus` y muestra banner cuando las metricas degradan por offline parcial.
- Target 3 queda cerrado para el alcance de Fase 2.
