# Technical Finding — Admin: edición de ficha y reapertura de ronda

**Context**: Expansión de capacidades del panel admin en CALAIRE-EA

## Los Milestones

### 1. Admin puede editar ficha F-PSEA-05A

La vista `/dashboard/rondas/[id]/participantes/[pid]/ficha` era un server component read-only. Se convirtió a un page + client component editable.

**Diferencia clave con el formulario del participante:**
- Admin no tiene restricción de `estado === 'borrador'` — puede editar fichas ya enviadas y de rondas cerradas
- `adminGuardarCampoFichaAction` hace `UPDATE ... WHERE id = fichaId` sin filtrar por estado
- El page usa `getOrCreateFicha(rpId)` para auto-provisionar si el participante no ha iniciado
- No existe botón de "Enviar ficha" — el estado de la ficha lo cambia solo el participante

### 2. Reabrir ronda cerrada

`reabrirRondaAction` en `dashboard/actions.ts` hace `cerrada → activa`. Validación: solo acepta rondas en estado `cerrada`. Flujo normal (`changeRondaStatusAction`) sigue igual: borrador → activa → cerrada. El botón en `StatusAction` es amber para distinguirlo visualmente.

## Archivos Creados

- `app/(protected)/dashboard/rondas/[id]/participantes/[pid]/ficha/actions.ts` — server actions admin
- `app/(protected)/dashboard/rondas/[id]/participantes/[pid]/ficha/FichaAdminEditor.tsx` — client component editable

## Archivos Modificados

- `app/(protected)/dashboard/rondas/[id]/participantes/[pid]/ficha/page.tsx` — usa `getOrCreateFicha` + renderiza `FichaAdminEditor`
- `app/(protected)/dashboard/actions.ts` — agrega `reabrirRondaAction`
- `app/(protected)/dashboard/page.tsx` — importa `reabrirRondaAction`, `StatusAction` muestra botón amber para cerradas

## Decisiones de diseño clave

- **Admin bypasses estado en DB**: `adminGuardarCampoFichaAction` omite `.eq('estado', 'borrador')` del UPDATE — edita cualquier ficha
- **Sin revalidatePath en admin editor**: misma estrategia que el formulario del participante — estado local en React
- **`getOrCreateFicha` en page admin**: si admin entra a la ruta de ficha de un participante que no la ha iniciado, se provisiona automáticamente en lugar de mostrar un mensaje de error
- **Reapertura separada de `changeRondaStatusAction`**: nueva action independiente en lugar de relajar las validaciones existentes, para preservar el flujo unidireccional del participante
