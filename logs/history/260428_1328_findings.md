# Technical Finding / Milestone

**Context**: Workflow de creación y configuración PT de rondas

## The Discovery

La entidad `rondaPtSampleGroups` sigue siendo necesaria para el modelo de datos y para el formulario de participante, porque las celdas PT se calculan como `ptItems.length * sampleGroups.length`. Sin embargo, el coordinador no necesita gestionar grupos de muestra manualmente para el flujo actual.

La solución aplicada fue mantener el grupo de muestra como detalle interno:

- Crear automáticamente un grupo `A` al crear la ronda.
- Asegurar también `A` al crear niveles PT, para cubrir rondas antiguas que no lo tengan.
- Quitar la UI de grupos de muestra.
- Considerar la configuración PT completa cuando hay niveles PT (`ptItems.length > 0`).

## Workflow Implementado

Después de crear una ronda:

1. Se crean analitos y participantes.
2. Se crea el grupo interno `A`.
3. La app redirige a `Configuración PT`.
4. El usuario carga niveles PT por bloque.
5. Solo entonces queda habilitada la gestión de participantes.
