# Technical Finding / Milestone

**Context**: Fase 2 / Targets 2-3 / UX offline SGC y participante

## The Discovery

El problema de banner offline era causado por doble captura: las paginas intentaban capturar errores offline, pero `safeConvexCall` ya los convertia en fallback vacio. La solucion fue agregar `safeConvexCallWithStatus`, que conserva el fallback y expone `offline` para que la UI pueda mostrar `BackendOfflineBanner`.

Tambien se confirmo que las rutas dinamicas no deben tratar fallback `null` como 404 cuando Convex esta caido. Las rutas ahora usan lecturas `WithStatus` para distinguir:

- `data: null, offline: false` -> recurso inexistente / `notFound()`.
- `data: null, offline: true` -> pantalla controlada con banner offline.
