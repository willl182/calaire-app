# Technical Finding: Fase 5 — Formulario Participante "Institutional Gold"

**Context**: Rediseño UI CALAIRE-EA — aplicar paleta dorada al formulario de entrada de datos del participante.

## The Discovery

### Cambios aplicados en `FormularioRonda.tsx`

1. **Progress bar dorada**: reemplazado `bg-blue-500` por `style={{ background: 'var(--pt-primary)' }}` y `var(--success)` al completar.
2. **KPI cards**: clases `card-accent` + `.numeric` en valores (Guardado / Pendientes / Observaciones).
3. **Inputs**: `focus:ring-blue-400` → `focus:ring-[rgba(253,185,19,0.4)]` en ambos inputs de réplicas e incertidumbre.
4. **Botón enviar**: `bg-blue-600 hover:bg-blue-700 ...` → clase `.btn-primary`.
5. **Pantalla éxito**: ícono `✓` en círculo `--pt-primary-subtle`, card con `.card-accent`.
6. **Hover de filas**: `hover:bg-gray-50` → handlers `onMouseEnter/Leave` con `var(--surface-muted)`.
7. **Fondo de página**: `bg-gray-50` → `var(--background)`.
8. **Cards de sección**: `bg-white rounded-xl shadow` → clase `.card`.

### Patrón establecido para inline styles vs clases Tailwind

- Colores de la paleta → siempre `style={{ color/background: 'var(--xxx)' }}` para respetar el sistema de diseño.
- Utilidades semánticas del proyecto (`.card`, `.btn-primary`, etc.) → clase CSS directamente.
- Colores Tailwind genéricos (grays, blues) → reemplazados por las CSS vars del sistema.
