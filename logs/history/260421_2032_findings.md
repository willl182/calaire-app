# Technical Finding / Milestone

**Context**: Cierre de Fase 0 para la integracion PT entre CALAIRE-EA y `pt_app`

## The Discovery

La Fase 0 quedo cerrada con dos decisiones que corrigen el supuesto equivocado del modelo actual:

- `replicate` en el CSV PT no representa la cantidad de replicas medidas; es un codigo entero estable por participante dentro de la ronda.
- La unidad minima de captura requerida por `pt_app` es `participante + contaminante + run + level + sample_group`.

Con esto se aprobo un modelo PT nuevo compuesto por:

- ampliacion de `ronda_participantes` con `participant_code`, `replicate_code` y `claimed_at`
- nueva tabla `ronda_pt_items` para `pollutant + run + level`
- nueva tabla `ronda_pt_sample_groups` para los grupos de muestra de la ronda
- nueva tabla `envios_pt` para la captura real compatible con el CSV objetivo
