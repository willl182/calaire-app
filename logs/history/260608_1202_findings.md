# Technical Finding: Cierre de documentacion Agent API

**Context**: Continuacion del trabajo de `/agent/v1/*` para dejar documentada y validada la API REST del agente.

## The Discovery

- Se creo `AGENT_API.md` con el flujo completo de claim (`POST /agent/auth` -> OTP -> `POST /agent/auth/claim/complete`), scopes, errores comunes y ejemplos curl.
- El endpoint generico ahora pasa query params como body logico para GET. Esto hace usable `GET /agent/v1/rondas/:id/sgc/snapshots?tipoRegistro=...`, que antes tenia `bodyFields` en el router pero el handler no leia body en GET.
- Para POST/PATCH/DELETE, el handler intenta JSON y conserva query params como fallback. Esto permite que DELETE con campos auxiliares funcione con body JSON o query params.
- Se elimino `lib/agent-api.ts`, un borrador incompleto no referenciado que rompia `pnpm lint`.
- Se reemplazo `FunctionReference<any>` en `lib/agent-router.ts` por un union tipado de query/mutation para cumplir ESLint.

## Key Files Affected
