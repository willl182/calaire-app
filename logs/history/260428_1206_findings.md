# Technical Finding: Phase 2 — Ronda Operative Summary Dashboard

**Context**: Coordinator Dashboard Redesign — Phase 2 Implementation

## The Discovery

The ronda summary page architecture composes **4 existing Convex queries in parallel** on the Next.js server to derive a single `RondaMetricas` object with 7 operative states. This avoids adding new Convex queries while providing a holistic view of round progress.

Key architectural decisions:

1. **Pure derivation function** — `derivarEstadoOperativo()` in `lib/rondas.ts` is a pure function (no I/O) that maps raw metrics to one of 7 operative states with a recommended action string. This makes it reusable in any context (dashboard, API, tests).

2. **Metric card variant derivation** — The page component computes visual variants (`success`/`warning`/`danger`/`default`) from metrics rather than from the operative state, allowing fine-grained visual feedback per metric.

3. **RondaContextNav as boundary** — Only the tab navigation component is `'use client'` (needs `usePathname()`). All page components remain Server Components, avoiding unnecessary client-side JavaScript.

4. **Two-step confirmation via searchParams** — State transitions (close/reopen) use `?confirm_cerrar=1` searchParams rather than client-side dialogs, keeping the entire flow server-rendered.
