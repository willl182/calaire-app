# Technical Finding: Consistencia Visual Dashboard CALAIRE-EA

**Context**: UI/UX — unificación de estilos entre páginas del dashboard

## The Discovery

### Patrón de layout consistente
Todas las páginas de contenido (listas, tablas, métricas) deben usar `max-w-7xl`. Solo las páginas de **formulario puro** (nueva ronda, ficha) pueden usar `max-w-3xl` — formularios anchos son mala UX.

### RondaContextNav como tarjeta integrada
El componente `RondaContextNav` no tenía clase CSS definida (`.ronda-context-nav` no existía en globals.css). Se reemplazó por una tarjeta `bg-[var(--surface)] border border-[var(--border)] rounded-xl shadow-sm` con breadcrumb arriba y tabs abajo — coherente con el lenguaje visual del resto del app.

### Regla anti-duplicidad de información
Si un dato ya está visible en una tarjeta KPI, **no** agregar un banner de alerta repitiendo lo mismo. Aplicado:
- "Config PT Pendiente" → tarjeta clicable, sin banner duplicado
- "Cupos sin reclamar" → contador en tarjeta, sin banner duplicado

### Orden lógico de los tabs de ronda
El flujo de trabajo es: configurar primero, luego invitar participantes. Por eso el orden correcto es:
**Resumen → Configuración PT → Participantes → Resultados**

### MetricaCard unificado
Antes usaba fondos de color por variante (rosa, gris, amarillo). Ahora usa `.card-accent` igual que la página de Resultados — borde izquierdo dorado, fondo blanco, `text-3xl numeric`.

## Key Files Affected

- `app/(protected)/dashboard/rondas/[id]/page.tsx` — MetricaCard, AlertasOperativas, orden de cards
- `app/(protected)/dashboard/rondas/[id]/RondaContextNav.tsx` — rediseño y reorden de tabs
- `app/(protected)/dashboard/rondas/[id]/configuracion-pt/page.tsx` — max-w-7xl
- `app/(protected)/dashboard/rondas/[id]/participantes/page.tsx` — max-w-7xl
