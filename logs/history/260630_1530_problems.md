## The Solution / Workaround

This error occurs when the Next.js Node.js server environment attempts to perform a server-side `fetch` query to the Convex backend while rendering the SgcPage, but the network request fails. 

**Common causes**:
1. **Convex Dev Server Offline**: The Convex dev server is not running locally (`pnpm exec convex dev` was not started or failed with a credential error).
2. **Incorrect URL**: `NEXT_PUBLIC_CONVEX_URL` in `.env.local` contains a typo, is undefined, or points to an offline deployment.
3. **Environment/Auth Block**: `pnpm exec convex dev` failed to authenticate (`TypeError: fetch failed` at boot/deploy time), leaving the project without generated config or active local deployment endpoint.

**Resolution Steps**:
1. Ensure `pnpm exec convex dev` is running successfully.
2. Confirm `NEXT_PUBLIC_CONVEX_URL` is correctly defined in `.env.local` and reachable from the host environment.

## Learnings to Preserve

Server-side queries using `ConvexHttpClient` in Next.js Server Components require a live connection to the Convex backend at render time. They will throw a blocking `TypeError: fetch failed` at runtime if the backend is unreachable.
