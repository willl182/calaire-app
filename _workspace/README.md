# Workspace interno

Esta carpeta agrupa material de trabajo que no es documentacion formal del aplicativo.

- `api/`: notas y guias operativas para integraciones internas.
- `auth/`: exploracion y pruebas de autenticacion.
- `plans/`: planes, roadmaps y propuestas de implementacion.
- `sgc/`: analisis y decisiones de trabajo sobre el modulo SGC.
- `guides/`: borradores o fuentes de guias de usuario.
- `reports/`: reportes de avance y actividad.
- `grills/`: revisiones, evaluaciones y contrastes de planes.

La documentacion vigente del aplicativo debe quedarse en `docs/`. El codigo de producto
debe quedarse en `app/`, `convex/`, `lib/`, `public/`, `scripts/` o `tests/`.
